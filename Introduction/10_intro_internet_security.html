— The Internet as an Attack Surface
How your browser and web activity create security risk
Every internet connection is a potential entry point for threats
When a device connects to the internet, it becomes reachable from billions of other connected devices worldwide — including those operated by attackers. While firewalls and other perimeter controls filter most unwanted inbound connections, internet-connected devices interact constantly with external systems through web browsing, email, application updates, cloud services, and other communication channels. Each of these interactions is a potential vector for a threat: a website may deliver malware, a download may contain hidden code, a browser extension may exfiltrate data, or a link may lead to a phishing page. Internet security is the practice of using the internet safely — taking advantage of its connectivity while minimizing exposure to its threats.
Web browsers are complex attack surfaces requiring careful security management
The web browser is the most-used application in most organizations, and it is also one of the most complex attack surfaces. Browsers execute code from every website visited — JavaScript, CSS, media — which creates opportunities for malicious sites to exploit browser vulnerabilities, conduct drive-by download attacks, or harvest data. Browser extensions add functionality but also add risk: a malicious or compromised extension can read all web traffic, capture passwords, and exfiltrate data silently. Browser security requires keeping the browser updated, limiting extensions to those specifically approved, and using security features like safe browsing that warn about known malicious sites. Work browsing should happen on managed, secured devices rather than personal devices with unknown extension configurations.
HTTPS verifies website identity and encrypts browser traffic
HTTPS — the padlock icon in the browser address bar — serves two functions: it encrypts the connection between your browser and the web server so that traffic cannot be read in transit, and it verifies that the server you are communicating with is actually the legitimate server for that domain (via the certificate system). HTTPS does not mean a website is safe or legitimate — attackers routinely use HTTPS on phishing sites because certificates are inexpensive and easy to obtain. What HTTPS tells you is that your connection to the site is encrypted and that the site has a certificate for the domain shown in the address bar. The domain itself is the meaningful indicator: 'microsoft.com' is legitimate; 'microsoft-support-login.com' is not, regardless of HTTPS.
Downloads carry significant risk — verify source and authenticity before executing
Every file downloaded from the internet is a potential malware delivery vehicle. Attackers create websites and file-sharing pages that appear to offer legitimate software, media, or documents but deliver malware. This includes fake download pages for popular software, torrent sites that bundle malware with requested content, and malicious documents delivered through cloud storage links. Before downloading and executing any file from the internet, verify that it comes from the official, legitimate source for that software or content. For software, this means the official vendor's website — not a third-party download aggregator. For documents, this means a source whose identity has been verified. Checking the file hash against the vendor's published value confirms that the file has not been tampered with in transit.
— Web-Based Threats
The attacks delivered through internet browsing and web applications
Malicious advertising (malvertising) delivers malware through legitimate ad networks
Malvertising is the use of online advertising networks to deliver malware or direct users to malicious sites. Attackers purchase advertising space through legitimate ad networks, submitting ads that contain hidden malicious code or that redirect to sites delivering drive-by downloads. Because the ads are served through legitimate networks onto legitimate websites — including high-traffic news sites, video platforms, and professional publications — users on reputable websites can be exposed to malicious content without the website operator's knowledge. Ad blockers reduce exposure to malvertising by preventing most advertising code from executing in the browser. For high-security environments, this represents a meaningful risk reduction for web browsing.
Credential harvesting sites: fake login pages that steal usernames and passwords
Credential harvesting through fake web pages is one of the most effective and commonly used attack techniques. A phishing email directs a user to a page that looks exactly like a legitimate service — Microsoft, Google, a banking site, a business application — where the user enters their credentials, which are immediately captured by the attacker. The pages are designed with meticulous attention to visual detail: the correct logo, the correct color scheme, the correct layout. The only reliable indicator that the page is fake is the domain in the browser's address bar. Developing the habit of checking the actual domain before entering any credentials — and never logging in through a link provided in an email — is the primary defense.
Watering hole attacks: compromising sites that the target organization's employees visit
A watering hole attack is a targeted variation in which an attacker identifies websites frequently visited by members of a specific organization or industry — professional publications, vendor portals, industry forums — and compromises those sites to deliver malware specifically to visitors from the target. Unlike broad phishing campaigns, watering hole attacks are designed to reach a specific audience who visits these sites as part of their professional routine. Detection is particularly difficult because the infection occurs on a legitimate site that the user has reason to trust and visit regularly. Current browser updates and behavioral endpoint detection are the primary defenses, as the user cannot reasonably avoid the compromised legitimate site.
Oversharing on professional and social networks aids attacker reconnaissance
Information shared on professional networking sites — job titles, employer, technical skills, colleagues, work history, project involvement, vendor relationships — is the raw material for spear phishing, social engineering, and targeted attacks. Attackers use publicly available information to craft convincing pretexts, identify high-value targets, map organizational structures, and discover technology systems in use. This does not mean professional networking should be abandoned — the benefits are real — but it does mean being thoughtful about what is shared publicly. Employees should be aware that their public professional profile is visible to potential attackers and should avoid disclosing specific internal system names, sensitive project details, or security-related role descriptions that provide targeting intelligence.
— Safe Browsing Practices
Habits that reduce risk while using the internet
Verify URLs before entering any information, especially credentials
The domain in the browser's address bar is the single most reliable indicator of whether you are on a legitimate site. Before entering any username, password, or sensitive information on a web page, verify the domain. Legitimate domains for major services are short and exact: 'google.com', 'microsoft.com', 'amazon.com'. Phishing domains use variations: adding words ('google-account-verify.com'), changing extensions ('.co' instead of '.com'), using hyphens ('micro-soft.com'), or using character substitutions ('rniicrosoft.com' with 'rn' instead of 'm'). These variations are designed to be overlooked in a quick reading. Password managers provide a useful secondary check: they store credentials by domain and will not autofill on a domain they do not recognize as the legitimate service.
Keep browsers and plugins updated — outdated browser software is a primary drive-by download vector
Browser vendors release security updates frequently because the browser's code executes content from every website visited, creating a large attack surface for vulnerabilities. Attackers closely track known browser vulnerabilities and target users who have not yet applied available patches. Enabling automatic browser updates ensures that security patches are applied as quickly as possible. Browser plugins — PDF readers, media players, Java — carry the same risk. Many browser vendors have moved toward built-in handling of common media types specifically to reduce plugin exposure. Extensions and plugins that are not actively used should be removed: they represent attack surface without ongoing benefit. Work browsers should be managed and updated by IT.
Use organizational web security controls — do not bypass them for convenience
Organizations implement web security controls — URL filtering, malware scanning, HTTPS inspection — to protect users from web-based threats. These controls filter out known malicious sites, scan downloads for malware, and provide visibility into web traffic that enables threat detection. Bypassing these controls — using personal mobile hotspots instead of corporate Wi-Fi, configuring browsers to use alternative DNS, or using proxy services to route around content filters — removes protective coverage that the controls provide and may create compliance violations. If a legitimate site is being blocked by organizational controls, the appropriate response is to request an exception through IT, not to work around the control. The controls exist because web-based threats are real, not as arbitrary restrictions.
Be cautious with browser extensions — they have extensive access to browser content
Browser extensions can significantly enhance browser functionality, but they operate with permissions that make malicious or compromised extensions extremely dangerous. An extension with permission to 'read and change all your data on the websites you visit' can capture passwords as they are typed, read sensitive information from any page, and transmit it to the extension's developer or to any server the extension communicates with. Extensions are routinely purchased by malicious actors after they have established a user base, then updated with malicious code. Limiting browser extensions to those specifically approved by IT, reviewing the permissions requested by any extension before installing, and periodically reviewing installed extensions to remove unused ones reduces this attack surface.
— Cloud and Remote Work Security
Internet security considerations for modern work patterns
Cloud applications require the same authentication security as internal systems
Cloud-hosted business applications — email platforms, collaboration tools, CRM systems, financial applications — are accessible from any internet connection, which means they face threats from any location in the world. Unlike on-premises systems behind a firewall, cloud applications are directly exposed to internet-based attacks. Enabling MFA on cloud applications is as critical as enabling it on internal systems — arguably more so, because the authentication endpoint is publicly accessible. Strong passwords, MFA, and monitoring for unusual login locations or times apply equally to cloud applications. Many cloud platforms provide security reports showing login locations and device types; periodically reviewing this information helps detect unauthorized access.
Remote work creates security considerations that differ from office environments
Working from home or other remote locations introduces security considerations that do not apply in a controlled office environment. The home network is typically less secure than the corporate network: home routers may have default passwords, firmware may not be updated, and family members may connect devices with varying security postures to the same network. Using the corporate VPN when accessing internal resources protects traffic on the home network. Maintaining physical security of the work area — closing windows and doors during sensitive calls, using privacy screens in public locations, and ensuring that work devices are not accessible to family members or household visitors — extends organizational security controls to the remote work environment.
Separating work and personal activity on work devices protects organizational data
Work devices are configured and maintained by IT to meet the organization's security requirements: specific security software, management tools, content filtering, and configuration standards. Using work devices for personal browsing, personal email, personal software downloads, and personal online accounts introduces risk: personal websites and services may have lower security standards, personal downloads may contain malware, and personal accounts that are compromised can become a pathway to work resources if they are accessed from the work device. Many organizations have acceptable use policies that define what personal use of work devices is permitted. Understanding and following these policies protects both the organization and the individual.
Recognize and avoid risky websites — not all internet destinations are equal
Some categories of websites carry significantly higher risk of malware delivery, drive-by downloads, and other threats: piracy and file-sharing sites, unofficial software download sites, sites offering 'free' versions of commercial software, and sites with excessive or aggressive advertising. These sites are frequently used to distribute malware because their operators are either deliberately malicious or lack the resources to vet the content and advertising they host. Organizational web filtering typically blocks access to these categories from managed devices. On personal devices used for occasional work, being aware of which site categories carry higher risk — and avoiding them when conducting work activity — reduces the probability of infection that could affect work accounts or data.
— Summary
Key concepts from this module
Every internet interaction is a potential threat vector — web browsing, downloads, and browser extensions all carry risk
HTTPS encrypts and authenticates the connection but does not make a site safe. The domain in the address bar is the reliable indicator of site legitimacy. Verify before entering any information.
Web-based threats include malvertising, credential harvesting pages, watering holes, and social profile reconnaissance
Each exploits a different aspect of internet use. Credential harvesting pages are defeated by checking the domain before logging in. Drive-by downloads are mitigated by current browser updates. Watering holes require behavioral endpoint protection.
Safe browsing: verify URLs, keep browsers updated, follow organizational controls, limit extensions
These four habits address the primary web threat vectors. Organizational controls provide protection that individual habits alone cannot — do not bypass them for convenience.
Cloud and remote work security: MFA on all cloud apps, VPN at home, separate work and personal activity
Cloud applications are internet-exposed and require the same authentication security as internal systems. Remote work environments require VPN use and physical security of work devices and conversations.

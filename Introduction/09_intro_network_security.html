— What Is a Network
How devices communicate and why the network is a security concern
A network connects devices so they can communicate and share resources
A computer network is a collection of devices — computers, servers, printers, phones, and other equipment — connected in a way that allows them to communicate with each other and share resources. The network is the infrastructure through which data moves: when you send an email, access a file on a shared drive, or visit a website, your data travels across a network. The network inside an organization is typically referred to as the local area network, or LAN. The internet is a global network of networks that allows communication between any two connected devices anywhere in the world. Networks are essential to virtually every modern business function — which also makes them an essential target for attackers.
Network traffic can be intercepted if communications are not encrypted
Data traveling across a network is transmitted as packets — small units of data that may pass through many intermediate devices (routers, switches, access points) on the way from source to destination. At each point of transit, the data can potentially be observed by anyone with access to that network segment. On an unencrypted network, traffic can be captured and read by any device on the same network segment — a technique called packet sniffing or network eavesdropping. This is why encryption of network communications is essential: HTTPS for web traffic, TLS for email, and encrypted VPN tunnels for remote access ensure that even if traffic is intercepted, the content cannot be read without the encryption key.
The network perimeter: separating trusted internal networks from untrusted external networks
The concept of a network perimeter defines the boundary between the organization's internal network — where devices are considered relatively trusted — and external networks, primarily the internet, where devices are untrusted. Firewalls enforce this boundary by inspecting traffic crossing the perimeter and applying rules about what is permitted. The traditional security model assumed that everything inside the perimeter was safe and everything outside was dangerous. This model is no longer adequate — attackers who breach the perimeter move laterally, cloud services extend data and applications beyond the perimeter, and remote workers connect from outside the perimeter. Modern network security goes beyond the perimeter to apply controls inside the network as well.
Network segmentation: dividing the network to limit the spread of threats
Network segmentation divides a single network into separate zones, with controlled access between zones. Rather than allowing any device on the internal network to communicate freely with any other device, segmentation isolates different types of systems: user workstations in one segment, servers in another, sensitive databases in a third, guest devices in a fourth. If an attacker compromises a device in the user segment, segmentation limits how far they can move — they cannot directly reach database servers in a separate segment without passing through access controls between zones. Segmentation is the network equivalent of compartmentalization: limiting the blast radius of any single compromise by restricting what a compromised device can reach.
— Common Network Threats
How attackers target and exploit networks
Man-in-the-middle attacks: intercepting communications between two parties
A man-in-the-middle (MITM) attack occurs when an attacker positions themselves between two communicating parties — intercepting, and potentially modifying, the communications without either party's knowledge. On a local network, ARP spoofing allows an attacker to redirect traffic meant for one device to their own device instead. On public Wi-Fi networks, rogue access points impersonate legitimate networks and capture all traffic from users who connect. Encrypted communications (HTTPS, TLS) prevent MITM attackers from reading the content of intercepted traffic, even if they can see that traffic is occurring. Certificate warnings in browsers — 'Your connection is not private' — may indicate a MITM attempt and should not be bypassed without understanding the cause.
Denial of service: overwhelming systems to make them unavailable
A denial of service (DoS) attack attempts to make a system or network service unavailable by overwhelming it with more requests than it can process. Distributed denial of service (DDoS) attacks use many thousands of compromised devices simultaneously to generate traffic volumes that exceed the target's capacity. DDoS attacks do not steal data — their goal is disruption. Organizations that depend on internet-facing services — web applications, customer portals, API endpoints — are at risk of DDoS attacks that make those services unavailable to legitimate users. DDoS mitigation services scrub malicious traffic before it reaches the organization's infrastructure. Availability is the CIA Triad property that DoS attacks specifically target.
Lateral movement: using network access to reach additional systems after initial compromise
Once an attacker has gained access to one device on a network, they typically attempt to use that access as a foundation for reaching additional systems — a process called lateral movement. This involves scanning the network for other devices, attempting to authenticate to those devices using credentials gathered from the compromised device or through credential attacks, and gradually expanding access to higher-value targets. Lateral movement is how a phishing attack against one employee's workstation can become a breach of a database server. Network segmentation, monitoring for unusual internal traffic patterns, and requiring strong authentication for internal system access are the primary defenses against lateral movement once initial compromise has occurred.
Rogue devices and unauthorized network access
Unauthorized devices on a network create security risks that bypass perimeter defenses entirely. A rogue wireless access point — an unauthorized Wi-Fi device connected to the internal network — allows attackers outside the building to access internal network resources without going through the organization's internet gateway and firewall. An employee who connects a personal device to the corporate network may introduce malware. A contractor who connects a laptop to an internal network port extends the organization's trust to a device outside its control. Network access control (NAC) systems authenticate devices before allowing them to join the network, ensuring that only approved and compliant devices can connect. Wireless network monitoring can detect rogue access points.
— Network Security Controls
The tools that protect networks from attack
Firewalls: the foundational perimeter control
A firewall is a network security device that monitors traffic between networks — typically between the internal network and the internet — and enforces rules about what traffic is allowed to pass. Basic firewalls filter traffic based on source address, destination address, and port. Next-generation firewalls (NGFW) add application-layer inspection, identifying and filtering specific applications and protocols regardless of what port they use, and intrusion prevention capabilities that detect and block known attack patterns. Every organization's internet connection should pass through a properly configured firewall. Firewall rules should follow the principle of least privilege: only the traffic necessary for business purposes should be permitted, with all other traffic denied by default.
VPN: encrypting remote access to internal resources
A virtual private network (VPN) creates an encrypted tunnel between a remote device and the organization's internal network, allowing remote workers to access internal resources as if they were physically on-site. VPN traffic is encrypted end-to-end, protecting it from interception on whatever network the remote user is connected to — including public Wi-Fi. Most organizations require employees to use a VPN when accessing internal systems from outside the office. Some organizations extend this requirement to all internet traffic when working remotely, routing everything through the organization's internet security controls rather than directly from the employee's home network. Following the organizational VPN policy when working remotely is an important security behavior.
Network monitoring: detecting threats that have bypassed preventive controls
No preventive control is perfect. Network monitoring provides visibility into what is happening on the network — allowing detection of threats that have bypassed firewalls, endpoint protection, and other controls. Network monitoring tools capture and analyze traffic patterns, flag unusual behaviors (large data transfers to unfamiliar destinations, connections to known malicious domains, internal traffic patterns consistent with lateral movement), and alert security teams to potential incidents. Security information and event management (SIEM) systems aggregate logs from network devices, endpoints, and applications and apply correlation rules to identify patterns that indicate attack activity. The speed of threat detection and response has a direct impact on the amount of damage an attack causes.
Wireless security: protecting Wi-Fi networks from unauthorized access
Wireless networks require specific security controls because their signal extends beyond physical walls — anyone within radio range can attempt to connect. WPA3 is the current standard for wireless network encryption and authentication; networks using older protocols (WEP, WPA) should be upgraded. Guest wireless networks should be isolated from the internal corporate network, preventing guests from accessing internal resources. Corporate wireless networks should require strong authentication — ideally certificate-based authentication tied to managed devices, rather than a shared passphrase that all employees know. Disabling wireless networks in areas where they are not needed and monitoring for rogue access points are additional wireless security practices.
— Safe Network Behaviors
Individual habits that reduce network-based risk
Public Wi-Fi is untrusted — use VPN when connecting from public networks
Public Wi-Fi networks — in coffee shops, hotels, airports, and conference venues — provide no guarantee of security. Other users on the same network may be able to intercept unencrypted traffic. The access point itself may be operated by an attacker posing as a legitimate network. Even on a legitimate public network, the operator cannot be held to the same security standards as a corporate network. When working from public Wi-Fi, connecting through the organization's VPN before accessing any business resources protects traffic in transit. For situations where the VPN is not available, limiting work to tasks that do not involve sensitive data, and avoiding logging in to sensitive accounts over public Wi-Fi, reduces exposure.
Never connect unknown devices to the corporate network
Connecting an unknown device — a found USB device, a personal laptop, a client's device — to the corporate network introduces a device that is outside the organization's security controls. The device may carry malware that spreads to other networked systems. It may have software that scans the network or captures credentials. It creates an unmanaged connection that bypasses the access controls designed to limit what can reach sensitive network resources. If a business need requires connecting an external device to the network, the appropriate path is through IT — who can assess the device, provide access through an isolated guest network, or establish appropriate controls before the connection is made.
Report unusual network activity to IT — slow systems, unexpected disconnections, unfamiliar devices
Network-based attacks often produce observable symptoms: unusual system slowness that may indicate a device being used for cryptomining or as part of a DDoS botnet, unexpected network disconnections that may indicate a MITM attack in progress, unfamiliar devices visible on the network, or applications attempting to connect to unfamiliar destinations. Users who notice these anomalies and report them to IT provide early warning that allows faster investigation and response. The security value of an employee who says 'my computer has been behaving strangely for the past two days' and reports it is significant — many incidents that were caught early were caught because a user noticed and reported something that seemed wrong.
Keep devices updated and use organizational security tools
Network security depends partly on the security of individual devices connected to the network. A single unpatched device with a known vulnerability can be exploited to gain a foothold on the network from which an attacker can move laterally. Keeping devices updated — applying operating system patches and application updates promptly — removes the vulnerabilities that network-based attacks use for lateral movement. Using organizational endpoint protection software rather than disabling it 'because it makes the computer slow' maintains the security coverage the organization requires. Devices that cannot meet the organization's security standards should not be connected to the corporate network — the NAC system enforces this technically, but understanding the reasoning supports cooperative security behavior.
— Summary
Key concepts from this module
Networks carry all organizational data — security of the network determines whether that data is protected in transit
Encryption of network communications (HTTPS, TLS, VPN) is the primary protection for data in motion. Unencrypted traffic on any network can be intercepted and read.
Firewalls, VPNs, segmentation, and monitoring are the core network security controls
Firewalls control what enters and exits. VPNs protect remote access. Segmentation limits lateral movement. Monitoring detects what preventive controls miss. Each addresses a different layer of network risk.
Public Wi-Fi is untrusted — always use VPN for business activity on public networks
The corporate VPN encrypts traffic regardless of what network you are on. Public networks cannot be trusted to protect your data. This is the most important individual network security behavior.
Report unusual network activity, keep devices patched, and never connect unauthorized devices
Users who notice and report anomalies provide early warning that enables faster response. Patched devices remove the vulnerabilities that lateral movement exploits. Unauthorized devices bypass the controls designed to protect the network.
